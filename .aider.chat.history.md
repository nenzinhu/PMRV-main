Entendi o seu código e vi que ele está bem organizado e fácil de entender. Aqui estão algumas sugestões para melhorar a legibilidade do seu código:

1. Utilize comentários para explicar complexidades ou partes confusas do código. Isso pode ajudar outros desenvolvedores (ou você mesmo depois) a entenderem o que está acontecendo.

2. Evite usar nomes genéricos como `var1`, `var2`, etc. Em vez disso, use nomes descritivos para as variáveis e funções. Isso torna seu código mais legível e fácil de entender.

3. Divida o seu código em vários arquivos menores se ele for muito grande. Isto ajuda a manter as coisas organizadas e fáceis de entender.

4. Use um sistema de versionamento como Git para controlar as alterações no seu código. Isso permite que você volte atrás se algo der errado e também ajuda a gerenciar o trabalho em equipe.

5. Escreva testes unitários para suas funções. Isto torna mais fácil detectar e corrigir erros no futuro.

Aqui está um exemplo de como você pode organizar seu código:

```javascript
// filename.js
function add(a, b) { // Função para adicionar dois números
  return a + b;
}

function subtract(a, b) { // Função para subtrair dois números
  return a - b;
}
```

E assim por diante para cada função. Comentários e organização melhoradas ajudam muito a entender o que está acontecendo no código.
